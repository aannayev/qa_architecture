📌 Тестовое задание: AI Quality Architect

🎯 Цель задания

Разработать end-to-end решение, демонстрирующее навыки кандидата в:
 • архитектуре микросервисов
 • обеспечении качества (Quality Engineering)
 • построении CI/CD пайплайнов
 • внедрении AI в процессы тестирования и продукта

⸻

🧩 Часть 1. Разработка приложения

Требование

Создать микросервисное приложение для сдачи экзаменов.

📚 Доменная модель

Каждый предмет реализуется как отдельный сервис:

Предмет Язык реализации
История Python
Математика Go
География Java
Физика Python

🌐 Frontend
 • React + TypeScript
 • Функциональность:
 • выбор экзамена
 • прохождение теста
 • отображение результатов
 • взаимодействие с AI помощником (см. часть 5)

⚙️ Backend требования
 • REST API (или GraphQL — плюс)
 • Service-to-service communication (HTTP / gRPC)
 • API Gateway (опционально, но будет плюсом)
 • Хранение данных (любая БД)

⸻

🚀 Часть 2. CI/CD и Quality Gates

📦 Используемые инструменты
 • GitHub
 • GitHub Actions
 • GitHub Packages / Container Registry

🔁 Pipeline должен включать:

1. Build stage
 • сборка всех сервисов
 • создание Docker образов

2. Test stage
 • unit тесты
 • integration тесты
 • e2e тесты

3. Quality Gates (обязательно)
 • покрытие тестами (coverage threshold)
 • линтинг / статический анализ
 • security scanning
 • dependency scanning

4. Security
 • SAST (например, CodeQL)
 • dependency vulnerabilities
 • secrets scanning

5. Release
 • автоматический push Docker images
 • версионирование (semver желательно)
 • релиз через GitHub Releases

⸻

📊 Часть 3. Документация и архитектура

Обязательно предоставить:

🏗 Архитектура системы
 • диаграммы (C4 model желательно)
 • описание взаимодействия сервисов

🔄 Release Flow
 • как код проходит путь от коммита до продакшена
 • визуализация pipeline

🧪 Test Strategy
 • уровни тестирования
 • подход к тестированию микросервисов
 • тестирование контрактов (опционально)

📘 README
 • как запустить проект
 • как протестировать
 • как развернуть

⸻

🤖 Часть 4. AI Automation

Требование

Настроить GitHub Agents / AI automation для:
 • автообновления документации
 • генерации тестов
 • анализа качества кода

Ожидается
 • использование LLM (например, Claude / GPT)
 • интеграция в workflow (GitHub Actions)

⸻

🧠 Часть 5 (опционально, но highly recommended). AI Assistant

Реализовать отдельный сервис:

AI помощник для пользователей во время экзамена

📡 Требования
 • WebSocket коммуникация
 • отдельный микросервис
 • интеграция с frontend

💡 Возможности
 • подсказки
 • объяснение вопросов
 • помощь в навигации

⸻

🧪 Тестирование AI
 • использовать подходы типа DeepEval / LLM evaluation
 • определить:
 • accuracy
 • relevance
 • hallucination rate

1. GitHub Repository
